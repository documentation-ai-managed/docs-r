## Upgrade Instructions

Follow these steps to update R in your project.

<Steps>
  <Step title="Check Current Version" icon="package">
    Run this command to view your installed version:
    ```bash
    r --version
    ```
  </Step>
  <Step title="Update Package" icon="download">
    <Tabs>
      <Tab title="npm" icon="nodejs">
        ```bash
        npm update r-docs@latest
        ```
      </Tab>
      <Tab title="yarn" icon="package">
        ```bash
        yarn upgrade r-docs@latest
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Verify Update" icon="check-circle">
    Restart your server and confirm the new version loads without issues.
  </Step>
</Steps>

## Recent Releases

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]} id="latest">

## New Features

- Added real-time collaboration for teams to edit docs simultaneously.
- Introduced version history with diff views for tracking changes.

## Improvements

- Enhanced search performance, now indexing `<100ms` for large sites.
- Better mobile responsiveness with improved touch navigation.

## Bug Fixes

- Fixed rendering issues with nested `{code blocks}` in MDX files.
- Resolved authentication errors during API key rotation.

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["bugfix", "improvement"]} id="previous">

## Improvements

- Optimized build times by 40% for static site generation.
- Added customizable brand colors, defaulting to `#3B82F6`.

## Bug Fixes

- Corrected table alignment problems in dark mode.
- Patched security vulnerability in webhook endpoints (CVE-2024-1234).

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["feature", "breaking"]} id="initial">

## New Features

- Core documentation engine with MDX v3 support.
- Integrated component library including `<Callout>`, `<Steps>`, and `<Tabs>`.
- Full-text search across all pages.

## Breaking Changes

- Migrated from v0.x config format; update your `r.config.js` file.

</Update>
