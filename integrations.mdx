## Git and Version Control Integration

Link your R workspace to Git repositories. You push changes directly from R to GitHub, GitLab, or Bitbucket. This keeps your documentation in sync with your codebase.

<Steps>
  <Step title="Connect Repository" icon="git-branch">
    In R settings, navigate to Integrations > Git. Enter your repository URL like `https://github.com/your-org/your-docs.git` and authenticate with a personal access token.
  </Step>
  <Step title="Commit Changes" icon="save">
    Edit your docs in R, then click Commit. Add a message and push to the main branch.
  </Step>
  <Step title="Pull Updates" icon="download">
    Pull the latest changes from Git to update your local workspace.
  </Step>
</Steps>

<Callout kind="tip">
  Use branch protection rules in Git to prevent accidental overwrites.
</Callout>

## API for Custom Automation

R exposes a REST API for programmatic access. You automate doc creation, updates, and queries using HTTP requests. Authenticate with `YOUR_API_KEY`.

<ParamField path="workspaceId" param-type="string" required="true">
  Your R workspace identifier.
</ParamField>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_KEY`.
</ParamField>

<Request tabs="JavaScript,cURL">
  ```javascript
  const response = await fetch('https://api.r-docs.com/v1/workspaces/{workspaceId}/pages', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      title: 'New Page',
      content: '# Hello from API'
    })
  });
  ```
  ```bash
  curl -X POST https://api.r-docs.com/v1/workspaces/{workspaceId}/pages \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "title": "New Page",
      "content": "# Hello from API"
    }'
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "id": "page-uuid-1234",
    "title": "New Page",
    "status": "created"
  }
  ```
</Response>

## Export and Import Functionalities

Export your documentation to various formats or import from external sources. You support Markdown, PDF, and HTML exports, plus CSV/JSON imports for bulk updates.

<Tabs>
  <Tab title="Export to Markdown" icon="file-text">
    Select pages, choose Markdown format, and download. Use this for Git commits.

    ```bash
    # Example CLI export (if available)
    r-docs export --workspace your-workspace --format md --output docs/
    ```
  </Tab>
  <Tab title="Import from GitHub" icon="github">
    Connect GitHub repo and import Markdown files automatically.

    <CodeGroup tabs="JavaScript,Python">
      ```javascript
      // Simulate import script
      await fetch('https://api.r-docs.com/v1/import/github', {
        method: 'POST',
        body: JSON.stringify({ repo: 'your-org/repo', branch: 'main' })
      });
      ```
      ```python
      import requests
      response = requests.post('https://api.r-docs.com/v1/import/github', json={
        'repo': 'your-org/repo',
        'branch': 'main'
      })
      ```
    </CodeGroup>
  </Tab>
  <Tab title="Bulk CSV Import" icon="upload">
    Prepare CSV with columns `title,content,path`. Upload via dashboard.
  </Tab>
</Tabs>

<Callout kind="info">
  Exports include frontmatter for seamless re-imports.
</Callout>

These integrations make R a flexible part of your workflow. Start with Git for collaboration, then explore API automations for scale.
